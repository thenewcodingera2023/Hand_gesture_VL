# Test Suite
# CREDITS TO ANTHROPIC CLAUDE OPUS 4.7 FOR MAKING THESE TESTS

Pytest-based tests for the hand gesture recognition pipeline. Each test file targets one source module; everything except two `@pytest.mark.slow` cases runs on synthetic data and finishes in seconds.

## Running

`pytest` is a development dependency only and is not in `requirements.txt`. Install it once into your environment:

```
pip install pytest
```

All fast tests:

```
python -m pytest -q -k "not slow"
```

Full suite, including slow tests that read `data/splits/test.npz` and `runs/mlp_best.pt`:

```
python -m pytest -q
```

A single file:

```
python -m pytest -q tests/test_model.py
```

A single test by name:

```
python -m pytest -q tests/test_evaluate.py::test_predict_batches_shape
```

None of the test files open the webcam. The live demo is exercised manually via `python -m src.inference` (see top-level `README.md`).

## Fixtures

- `tests/conftest.py` ÔÇö shared pytest fixtures (synthetic feature tensors, tmp paths). Most test files pull from here rather than rebuilding inputs.
- Slow-marked tests use a session-scoped fixture that prefers `data/splits/` when present.

## Test Files by Stage

### Stage 1 ÔÇö Per-hand preprocessing, feature assembly, synthetic builder

- **`tests/test_preprocessor.py`** ÔÇö covers `src/preprocessor.py`. Wrist-at-origin, palm scale = 1, left-hand mirroring flips `x` only, z-padding 2D Ôćĺ 3D, bone vectors, finger extension ratios in `[0, 1]`, pairwise fingertip distances, output shape `(138,)`.
- **`tests/test_feature_assembler.py`** ÔÇö covers `src/feature_assembler.py`. No-hand all-zeros, right-hand-only, left-hand-only, and two-hand assembly. Verifies the canonical `(279,)` layout: presence flags at indices `RIGHT_PRESENT_IDX` / `LEFT_PRESENT_IDX`, inter-hand distance at `INTER_DIST_IDX`, and feature slices `RIGHT_FEAT_SLICE` / `LEFT_FEAT_SLICE`.
- **`tests/test_synthetic_builder.py`** ÔÇö covers `src/synthetic_builder.py`. Synthetic count-class output shape `(n, 279)`, presence flags `[1, 1]`, inter-hand distance sampling from the empirical prior, deterministic seeding.

### Stage 2 ÔÇö Dataset splits and augmentation

- **`tests/test_dataset.py`** ÔÇö 29 tests (27 fast + 2 `@pytest.mark.slow`) covering `src/dataset.py`. No `user_id` leakage between splits (real **and** synthetic source users via the `right_src_idx` / `left_src_idx` traceback), shape and dtype invariants, every class appears in every split, oversampling restricted to train with counts 6ÔÇô18 reaching the 5,000 target, ~80 / 10 / 10 real-row proportions, presence-flag invariants, augmented-row finiteness, determinism of `split_users`, `augment_features` zero-sigma identity, `oversample_train_counts` target hit. The slow tests rebuild the full splits into a temp dir twice to verify byte-identical determinism and seed sensitivity.

### Stage 3 ÔÇö Baselines

- **`tests/test_baseline.py`** ÔÇö 24 tests covering `src/models/baseline.py` on a synthetic 28-class 279-d dataset generated by `sklearn.datasets.make_classification`. Schema validation rejects wrong dim / dtype / NaN / out-of-range labels / length mismatch / missing `X` or `y`; `load_label_ids` matches `data/labels.json`; pipeline factories produce `StandardScaler + LogisticRegression / SVC` with the spec hyperparameters; LR/SVM training returns fitted estimators; stratified subsampling is deterministic and class-preserving; the scaler is fit only on train (no leakage); `evaluate_classifier` returns the six expected metrics in `[0, 1]` with `merged_accuracy Ôëą accuracy`; label-equivalence merging works; CSV writers create parent dirs, append without duplicating headers, and reject rows with missing required keys; end-to-end smoke through `joblib.dump` / `joblib.load`.

### Stage 4 ÔÇö MLP and training plumbing

- **`tests/test_model.py`** ÔÇö 22 tests covering `src/models/mlp.py` and the training helpers in `src/train.py`. Synthetic data only ÔÇö never touches `data/splits/*.npz`. Asserts `GestureMLP(torch.randn(8, 279)).shape == (8, 28)`, logits are finite, `softmax(logits).sum(-1) Ôëł 1` in eval mode, `nn.CrossEntropyLoss` consumes logits directly, single Adam step changes `linear_layers[0].weight`, `compute_grad_norm` is `0` before backward and `> 0` after, `compute_layer_weight_norms` returns 4 finite positive values, Kaiming-uniform init keeps `weight.abs().mean()` inside `[0, sqrt(6/fan_in))`, all biases zero at construction, dropout makes train-mode forwards differ but eval-mode forwards identical, loss strictly decreases over 30 Adam steps, constructor rejects wrong-length `hidden_dims` / `dropouts` and out-of-range dropout probabilities, `build_dataloaders` enforces `float32` `X` / `int64` `y` / feature dim 279, `append_log_row` enforces the 16-column header and rejects missing keys, `save_checkpoint` round-trips via `torch.load(..., weights_only=False)` to identical model output.

### Stage 5 ÔÇö Real-time inference helpers

- **`tests/test_smoother.py`** ÔÇö covers `src/smoother.py`. Pure-logic tests: sliding-window majority vote, confidence gate, no-hand reset after 5 consecutive empty frames, idle state returns `"---"`.
- **`tests/test_inference.py`** ÔÇö covers `src/inference.py` helper functions only (`build_feature_vector`, `load_inference_artifacts`, `predict_probs`). The webcam loop itself is **not** exercised by pytest.

### Stage 6 ÔÇö Evaluation and MV visualization

- **`tests/test_evaluate.py`** ÔÇö 19 tests covering `src/evaluate.py` on synthetic NPZ + a freshly built checkpoint saved via `train.save_checkpoint` to `tmp_path`; never reads `data/splits/*.npz` or `runs/mlp_best.pt`. Covers schema validation (missing `X`, wrong feature dim, invalid label IDs), checkpoint loading (eval-mode model, scaler shape `(279,)`, label map covers 0..27, dim-mismatch rejected, missing-file rejected), `predict_batches` shapes + softmax sums, classification metrics range and zero-support handling, latency timing positivity, both confusion-matrix PNGs written, JSON round-trip with `NaN Ôćĺ null`, per-class CSV schema, end-to-end `run_evaluation` writes all six artefacts.
- **`tests/test_mv_visualization.py`** ÔÇö 15 fast tests + 1 `@pytest.mark.slow` covering `src/mv_visualization.py`. Covers loss-surface grid shape and weight restoration after the sweep (in-place mutation in a `try/finally`), bad-arg rejection, `plot_loss_surface` PNG write, `plot_gradient_norm` requires the `grad_norm` column, `compute_input_pca` shape / determinism / training-scaler usage, `select_active_first_layer_neuron` determinism, controlled toy-model chain-rule matches autograd to `1e-6`, real-architecture chain-rule with random-init `GestureMLP` (5 samples, at least one gate-active, `abs_error < 1e-5`), `verify_chain_rule_batch` requires Ôëą 10 / 12 passes, raises when tolerance is forced to `1e-30`, single-sample shape validation. The slow test exercises the real `runs/mlp_best.pt` + `data/splits/test.npz` and asserts Ôëą 10 / 12 pass + Ôëą 8 / 12 gate-active.

## Disk vs Pure Unit

| Category | Tests |
|---|---|
| Pure unit (no disk) | `test_preprocessor`, `test_feature_assembler`, `test_synthetic_builder`, `test_smoother`, `test_inference` |
| Tmp-disk only (uses `tmp_path` fixtures) | `test_baseline`, `test_model`, `test_evaluate`, fast tests in `test_dataset` and `test_mv_visualization` |
| Reads `data/splits/` / `runs/mlp_best.pt` | `@pytest.mark.slow` tests in `test_dataset.py` and `test_mv_visualization.py` |
| Requires webcam | none ÔÇö live demo is manual |
